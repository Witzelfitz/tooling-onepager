03 · SFTP 15 Min

SFTP Konfiguration (Copy / Paste)

Du wirst hier Schritt für Schritt geführt – damit SFTP im richtigen Projektordner sofort sauber funktioniert.

Was heisst „Projektordner öffnen“?
  • Du öffnest in VS Code genau den Ordner deiner Website (z. B. im1-portfolio-max).
  • Darin liegen deine Dateien wie index.html, styles.css, script.js.
  • Nicht Desktop/Downloads/ganzer Benutzerordner öffnen.
  • Quick-Check: In VS Code links im Explorer siehst du oben den Projektnamen.
SFTP-Datei im Projekt erstellen (genau so)
  1. Projektordner öffnen in VS Code (nicht den Desktop, nicht Downloads).
  2. Command Palette öffnen: CMD + SHIFT + P (Mac) / CTRL + SHIFT + P (Windows).
  3. Tippen: SFTP: Config und ausführen.
  4. VS Code erstellt automatisch den Ordner .vscode mit der Datei sftp.json.
  5. Den gesamten Inhalt unten einfügen, Platzhalter ersetzen, speichern.

🔴 Pflicht-Schritt: Ohne diese SFTP-Konfiguration funktioniert dein Upload/Deploy nicht zuverlässig.

{
  "name": "hostpoint-main",
  "host": "replace-this-hostname",
  "protocol": "ftp",
  "port": 21,
  "username": "replace-this-username",
  "password": "replace-this-password",
  "remotePath": "/",
  "uploadOnSave": true,
  "ignore": [
    ".vscode",
    ".git"
  ]
}
Sauber arbeiten mit SFTP
  • Immer im richtigen Projektordner arbeiten.
  • Vor erstem Sync: kleine Testdatei deployen und URL checken.
  • Nie fremde/alte Projekte überschreiben (remotePath prüfen).
  • Credentials niemals in README/Code committen.
04 · GitHub 20 Min

GitHub Account + GitHub Desktop

FHGR Mail für Education Benefits nutzen, danach in GitHub Desktop ein neues Projekt sauber aufsetzen.

↗ GitHub Education
↗ GitHub Desktop

Projekt in GitHub Desktop erstellen (klick-genau)
  1. GitHub Desktop öffnen
  2. Add ▾ klicken
  3. Create New Repository... wählen
  4. Repository Name setzen (z. B. im1-portfolio-max-muster)
  5. Local Path wählen, der zu deinem Modul-Projekt gehört
  6. Create Repository klicken
  7. Publish repository klicken
Repository nach Abgabe auf Public stellen (optional, wenn gefordert)
  1. GitHub im Browser öffnen und ins richtige Repo gehen.
  2. Settings → ganz nach unten zu Danger Zone.
  3. Change repository visibility wählen.
  4. Auf Make public stellen und bestätigen.
  5. Vorher unbedingt Secrets/Passwörter prüfen.

🔴 Pflicht-Schritt: .gitignore schützt dich vor peinlichen/leaken Dateien im Repository.

.DS_Store
config.php
.vscode
.env
node_modules
Sauber arbeiten mit GitHub (Pflicht)
  • Commit-Historie in kleinen, verständlichen Schritten.
  • Klare Commit-Messages (feat:, fix:, chore:).
  • Keine Secrets/API-Keys im Repo.
  • Vor Public-Schaltung immer Secret-Check machen.
05 · Hostpoint 15 Min

Hostpoint Zugang + FTP Credentials holen

Login meist mit vorname.nachname@stud.fhgr.ch, danach FTP Daten in sftp.json eintragen.

Hostpoint Klick-Checklist (genaue Reihenfolge)
  1. Login mit vorname.nachname@stud.fhgr.ch
  2. Hosting / richtigen Service öffnen
  3. Server Bereich öffnen
  4. FTP bzw. FTP Accounts öffnen
  5. Hostname, Username, Passwort kopieren
  6. Daten in .vscode/sftp.json einsetzen und speichern
  7. In VS Code: SFTP: Sync Local -> Remote ausführen
Kurz-Ausflug: Subdomains (z. B. projektname.deinedomain.ch)
  • Subdomain in Hostpoint unter Domains/Zone verwalten erstellen.
  • Subdomain auf den korrekten Webpfad zeigen lassen.
  • Danach Upload testen und Browser-Cache einmal leeren.
  • Für Modulprojekte: lieber kurze, klare Subdomain-Namen nutzen.
06 · Mac / Windows 8 Min

MacBook Lane vs. Windows Lane

Damit niemand bei Shortcuts oder Menüs hängen bleibt: beide Wege klar nebeneinander.

🍎 MacBook Lane
  • Command Palette: CMD + SHIFT + P
  • Datei speichern: CMD + S
  • Suche im Projekt: CMD + SHIFT + F
  • SFTP Sync: Command Palette → SFTP: Sync Local -> Remote
  • GitHub Desktop: Publish + Commit aus dem App-Fenster
🪟 Windows Lane
  • Command Palette: CTRL + SHIFT + P
  • Datei speichern: CTRL + S
  • Suche im Projekt: CTRL + SHIFT + F
  • SFTP Sync: Command Palette → SFTP: Sync Local -> Remote
  • GitHub Desktop: Publish + Commit aus dem App-Fenster
Wichtigste Punkte für beide Lanes
  • Immer zuerst den korrekten Projektordner öffnen.
  • Kein Copy-Paste von Passwörtern in Code-Dateien.
  • Vor Abgabe: Repo aufräumen, letzte Live-URL prüfen, dann erst final committen.
